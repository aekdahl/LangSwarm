LangSwarm uses the **Model Context Protocol (MCP)** as its universal standard for connecting agents to external data and systems.

Instead of writing custom python functions for every tool, you can connect your agents to any **MCP Server**‚Äîwhether it's running locally on your machine or hosted remotely.

## üöÄ Quick Start

Connect an agent to the `filesystem` and `sqlite` tools in seconds using `uvx`.

```python
from langswarm.core.agents import AgentBuilder

agent = await (AgentBuilder("researcher")
    .openai()
    .model("gpt-4o")
    
    # 1. Connect to Local Filesystem
    .add_mcp_server(
        name="fs",
        command="uvx", 
        args=["mcp-server-filesystem", "/Users/alex/projects"]
    )
    
    # 2. Connect to Remote API (SSE)
    .add_mcp_server(
        name="api",
        url="https://api.weather.com/mcp"
    )
    .build())

# Agent can now read files and check weather!
result = await agent.chat("Read analysis.py and check the weather in NYC")
```

## üß© Standard Tools

LangSwarm works instantly with the growing ecosystem of standard MCP servers:

*   **Filesystem**: Read/write access to safe directories.
*   **Postgres**: Query and manage SQL databases.
*   **Git**: Commit, push, and manage repositories.
*   **Slack/Discord**: Send messages and read channels.
*   **Memory**: Advanced vector search and knowledge graphs.

*Check out the [Official MCP Server Registry](https://github.com/modelcontextprotocol/servers) for more.*
